### Submission Requirements

#### 🟦 Track 1 — Trading Agent

* **Demo link (required)** — publicly accessible, must actually run; include backtest or sim trading records; no real capital required
* **Project description (required)** — under 200 words: what problem it solves, strategy loop overview (perception → decision → execution → risk management); which Bitget AI modules used
* **Demo video (optional)** — no longer than 3 minutes

#### 🟩 Track 2 — Trading Infra

* **GitHub repository link (required)** — runnable code with complete README; other developers can integrate with low friction
* **Demo video (required)** — no longer than 3 minutes, showing core functionality and integration
* **Project description (required)** — what problem it solves, technical approach, extensibility notes

#### 🟧 Track 3 — US Stock AI Trading

* **Demo link or GitHub repository (one required)**
* **Project description (required)** — under 200 words: does it solve a real problem in the US stock tokenized trading space
* **Demo video (optional)** — no longer than 3 minutes

### Community Post Requirements (All Tracks)

* Repost the Bitget [official interaction post](https://x.com/Bitget_AI/status/2061719206106919039?s=20) and publish your project intro tagged **#BitgetHackathon** and @ the Bitget AI official account, then include the post links in your submission — qualifies for the Community Impact Award (500 USDT × 3).&#x20;
* Submit a runnable Demo + qualifying post（ (a development diary showcasing your project) ） to also earn the Participation Award (+50 USDT per team).

### Judging Criteria

**No fixed formula. The projects that stand out solve a real problem and actually run.**

Track-specific focus:

* **Track 1** — Complete strategy loop (perception → decision → execution → risk management); validated through backtest or sim trading
* **Track 2** — Can other developers actually integrate it with low friction; genuinely solves a pain point rather than reproducing existing tools
* **Track 3** — Does it solve a real problem in the US stock tokenized trading space; verifiable backtest or sim trading records; uses Bitget's US stock data or tools

⚠️ Baseline for all tracks: Demo must be real and runnable; clear answer to what problem it solves; at least one form of verifiable usage evidence (real/sim trade logs, API call volume, or user count).

### [Bitget Agent Hub](https://github.com/BitgetLimited/agent_hub) (Crypto & Stocks Agent's Trading Arsenal)

Bitget Agent Hub is a trading tool platform built for AI developers. Three layers, use what you need:

<table><thead><tr><th width="135.53125">Module</th><th width="283.09375">What it is</th><th>How it helps</th></tr></thead><tbody><tr><td><strong>Tools</strong></td><td>58 trading APIs (spot, futures, account, etc.)</td><td>Your Agent can place orders, check positions, and manage assets without writing any API wrappers</td></tr><tr><td><strong>Skill Hub</strong></td><td>5 analyst-grade perception Skills (macro, sentiment, technicals, on-chain, news)</td><td>Give your Agent market awareness in minutes — just configure and go</td></tr><tr><td><strong>MCP Server</strong></td><td>One-line setup for Claude / Cursor / Codex</td><td>Call all Bitget trading capabilities directly from the AI tools you already use</td></tr></tbody></table>

**Quick install:**

```bash
npx bitget-hub upgrade-all --target claude
```

Deploy to specific tools:

```bash
npx bitget-hub install --target codex
npx bitget-hub install --target claude,codex
```

**Configure Bitget API Key:** Log in to bitget.com → Settings → API Management, create a key with Read and/or Trade permissions:

```bash
export BITGET_API_KEY="your-api-key"
export BITGET_SECRET_KEY="your-secret-key"
export BITGET_PASSPHRASE="your-passphrase"
```

**Connect via MCP Server (Claude Code / Cursor):**

```bash
claude mcp add -s user \
  --env BITGET_API_KEY=your-api-key \
  --env BITGET_SECRET_KEY=your-secret-key \
  --env BITGET_PASSPHRASE=your-passphrase \
  bitget \
  -- npx -y bitget-mcp-server
```

## 5. FAQ

<details>

<summary>Q: Can I participate without trading experience?</summary>

Yes. The [Bitget Agent Hub GitHub repository](https://github.com/BitgetLimited/agent_hub) provides a complete skeleton — just extend it with your own ideas. Bitget Playbook also lets you run a trading strategy end-to-end without writing code. **No real capital required** — sim trading and backtest records are accepted.

</details>

<details>

<summary>Q: Can I participate solo? How does team registration work?</summary>

Absolutely. Solo and team entries are judged equally. Everyone registers as an individual by default — no need to submit teammate information at registration. Final team details will be collected again at the submission stage. Teams can have up to 5 members. Looking for teammates? Post in the official Telegram community.

</details>

<details>

<summary>Q: What if my registration fails?</summary>

Contact an admin directly in the [official Telegram community](https://t.me/+o1tYqQ_lXxllYjgy).

</details>

<details>

<summary>Q: How do I use Bitget Agent Hub?</summary>

Three ways, use what fits:

* **MCP Server**: one command to connect Claude Code / Cursor; then call 58 Bitget trading APIs through natural language
* **Skill Hub**: 5 plug-and-play perception modules (macro / on-chain / sentiment / technicals / news) — configure and go
* **Direct tool use**: clone from GitHub for fully custom setups

Detailed setup in Chapter 4.

</details>

<details>

<summary>Q: How do I use Bitget Playbook?</summary>

Two manual steps first (create sub-account with your registered UID + contact admin via Telegram to get a Playbook API Key), then paste a single prompt into Claude Code — the Agent handles install → strategy creation → backtest → publish automatically. Detailed steps in Chapter 4.

</details>

<details>

<summary>Q: How do I claim MuleRun Credits? What if I didn't receive them?</summary>

Join the official Telegram community, sign up on MuleRun, and enter code `0526BITGET` at [credits.mule.page](https://credits.mule.page/) to claim 2,000 Credits. One claim per email. If you didn't receive them, DM an admin with your UID.

</details>

<details>

<summary>Q: How do I claim Qwen token credits? What if I didn't receive them?</summary>

The first 500 registered teams receive an API Key by email within 24h of registration. It works with Cursor and Codex (**Claude Code not supported**). If you didn't receive it, @ an admin in the official Telegram community with your UID and registration time.

</details>

<details>

<summary>Q: The Qwen API Key doesn't work / can't connect to Qwen. What do I do?</summary>

This key routes through a Bitget proxy — it **cannot connect directly to the official Qwen API**. You must configure the correct Base URL (`https://hackathon.bitgetops.com/v1`) as shown in the Qwen Setup Guide in Chapter 4. Alternatively, use your own Qwen account API Key.

</details>

<details>

<summary>Q: How do I qualify for the Community Impact Award?</summary>

Repost the Bitget [official interaction post](https://x.com/Bitget_AI/status/2061719206106919039?s=20) and publish your own content (project ideas, progress, Showcase, etc.) tagged **#BitgetHackathon** and @ the Bitget AI official account. Submit the post links with your project. Submitting a runnable Demo + qualifying post also earns the Participation Award (+50 USDT per team).

</details>

<details>

<summary>Q: Will connecting Agent Hub with my own API Key trigger auto-trades?</summary>

Currently only read access is supported — it syncs positions to a simulated account. Order execution is not fully implemented yet; no real-account trades will be triggered automatically.

</details>

<details>

<summary>Q: What if I hit a technical issue?</summary>

Join the [official Telegram community](https://t.me/+o1tYqQ_lXxllYjgy) — the technical team is online throughout for support.

</details>
